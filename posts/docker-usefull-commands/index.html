The most useful Docker commands

By Jerbi Mohamed Chamseddine
Posted 2025-02-07 Updated 2025-02-22 3 min read

Here’s a comprehensive list of the most useful Docker commands for managing containers, images, networks, and volumes. These commands are essential for working with Docker in development, testing, and production environments.

Docker Basics

  1. Check Docker Version:
    1

    docker --version
  2. Display System-Wide Information:
    1

    docker info

Working with Images

  1. Pull an Image from Docker Hub:
    1

    docker pull <image_name>:<tag>

    Example:

    1

    docker pull nginx:latest
  2. List Downloaded Images:
    1

    docker images
  3. Remove an Image:
    1

    docker rmi <image_name>:<tag>
  4. Build an Image from a Dockerfile:
    1

    docker build -t <image_name>:<tag> .

    Example:

    1

    docker build -t myapp:1.0 .
  5. Tag an Image:
    1

    docker tag <image_id> <new_image_name>:<tag>
  6. Push an Image to Docker Hub:
    1

    docker push <image_name>:<tag>

Working with Containers

  1. Run a Container:
    1

    docker run <image_name>:<tag>

    Example:

    1

    docker run -d -p 8080:80 nginx:latest
  2. Run a Container in Interactive Mode:
    1

    docker run -it <image_name>:<tag> /bin/bash
  3. List Running Containers:
    1

    docker ps
  4. List All Containers (Running and Stopped):
    1

    docker ps -a
  5. Stop a Running Container:
    1

    docker stop <container_id>
  6. Start a Stopped Container:
    1

    docker start <container_id>
  7. Restart a Container:
    1

    docker restart <container_id>
  8. Remove a Stopped Container:
    1

    docker rm <container_id>
  9. Force Remove a Running Container:
    1

    docker rm -f <container_id>
  10. View Container Logs:
    1

    docker logs <container_id>
  11. Follow Container Logs:
    1

    docker logs -f <container_id>
  12. Execute a Command in a Running Container:
    1

    docker exec -it <container_id> <command>

    Example:

    1

    docker exec -it mycontainer /bin/bash
  13. Inspect a Container:
    1

    docker inspect <container_id>
  14. Copy Files Between Host and Container:
    1
2

    docker cp <file_path> <container_id>:<container_path>
docker cp <container_id>:<container_path> <file_path>
  15. Pause/Unpause a Container:
    1
2

    docker pause <container_id>
docker unpause <container_id>

Networking

  1. List Networks:
    1

    docker network ls
  2. Create a Network:
    1

    docker network create <network_name>
  3. Inspect a Network:
    1

    docker network inspect <network_name>
  4. Connect a Container to a Network:
    1

    docker network connect <network_name> <container_id>
  5. Disconnect a Container from a Network:
    1

    docker network disconnect <network_name> <container_id>
  6. Remove a Network:
    1

    docker network rm <network_name>

Volumes

  1. List Volumes:
    1

    docker volume ls
  2. Create a Volume:
    1

    docker volume create <volume_name>
  3. Inspect a Volume:
    1

    docker volume inspect <volume_name>
  4. Remove a Volume:
    1

    docker volume rm <volume_name>
  5. Mount a Volume in a Container:
    1

    docker run -v <volume_name>:<container_path> <image_name>

Docker Compose

  1. Start Services:
    1

    docker-compose up
  2. Start Services in Detached Mode:
    1

    docker-compose up -d
  3. Stop Services:
    1

    docker-compose down
  4. Rebuild and Restart Services:
    1

    docker-compose up --build
  5. View Logs for a Service:
    1

    docker-compose logs <service_name>
  6. Scale Services:
    1

    docker-compose up --scale <service_name>=<num_instances>

System Cleanup

  1. Remove All Stopped Containers:
    1

    docker container prune
  2. Remove All Unused Images:
    1

    docker image prune -a
  3. Remove All Unused Networks:
    1

    docker network prune
  4. Remove All Unused Volumes:
    1

    docker volume prune
  5. Remove All Unused Objects:
    1

    docker system prune -a

Advanced Commands

  1. Save an Image to a Tar File:
    1

    docker save -o <file_name>.tar <image_name>
  2. Load an Image from a Tar File:
    1

    docker load -i <file_name>.tar
  3. View Resource Usage:
    1

    docker stats
  4. View Docker Events:
    1

    docker events
  5. Check Disk Usage:
    1

    docker system df

Docker Swarm (For Orchestration)

  1. Initialize a Swarm:
    1

    docker swarm init
  2. Join a Swarm as a Worker:
    1

    docker swarm join --token <worker_token> <manager_ip>:<port>
  3. List Nodes in the Swarm:
    1

    docker node ls
  4. Deploy a Stack:
    1

    docker stack deploy -c <compose_file> <stack_name>
  5. List Services in a Stack:
    1

    docker stack services <stack_name>

This list covers the most useful Docker commands for day-to-day operations. Let me know if you need further clarification or examples! 🐳🚀
